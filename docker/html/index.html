Welcome to Fileglancer

To start using Fileglancerlogin here with your Janelia credentials (same as for the cluster)
When you login into Fileglancer, you will be asked to choose one of the following job profiles for your JupyterLab instance:
  • Local server
    You can start your Fileglancer instance on this server for free compute. This server has 128 cores and 500 GB of RAM, so feel free to use some threading or load a 20 GB image into memory -- just be reasonable about sharing the server.
  • Cluster
    There are several profiles which launch your Fileglancer instance on the compute cluster, with access to more computing power including GPUs. To use this mode, you must have access to the Janelia Compute Cluster which can be requested through the Help Desk. If you would like to request a new cluster profile, contact Konrad Rokicki on Slack.

Using Conda

If you have an existing Conda environment you'd like to use in a notebook, just make it available to Fileglancer like this: 
    conda install ipykernel
    python -m ipykernel install --user --name myenv --display-name "My Environment Name"

If you are using the "Local server" profile, you can create your own conda environment by using File->New->Terminal and then typing these commands (substituting your own environment names). 

    eval "$(/usr/bin/conda shell.bash hook)"
    /usr/bin/conda create --name myenv python=3.8 ipykernel
    /usr/bin/conda activate myenv
    python -m ipykernel install --user --name myenv --display-name "My Environment Name"
Restart your kernel and you should now see your new conda environment when creating a new Notebook.
